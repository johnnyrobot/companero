# Course Companion (companero) — PWA

[![CI](https://github.com/johnnyrobot/companero/actions/workflows/ci.yml/badge.svg)](https://github.com/johnnyrobot/companero/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

Minimal, accessible Progressive Web App for students to store class info locally on their device. No accounts. No cloud. Works offline.

## Features
- **Local-only storage** (browser `localStorage`) — no backend, no network calls for user data
- Add/edit/delete classes with name, section, days/times, location, notes
- Reorder classes; search/filter
- Export/Import backup (JSON)
- Installable PWA with **reliable offline + update behavior** (content-hashed assets, network-first navigations)
- Accessible, non-blocking modal dialogs (focus trap, ARIA labelling, focus restore)
- Hardened edge: CSP + security headers, immutable caching only for fingerprinted assets

## Develop

Requires **Node ≥ 20** (uses the built-in `node:test` runner). Dev/build deps only — the served image has **no production dependencies**.

```bash
npm install        # devDeps: jsdom (tests), sharp (icon generation)
npm test           # run the unit suite (must stay green)
npm run build      # content-hash assets into dist/ (rewrites refs, derives SW CACHE_NAME)
npm run icons      # regenerate the PNG app icons (only needed on a rebrand)
```

`build.mjs` is a pure-Node build: it fingerprints `app.js` / `styles.css` / `translations.js` + icons into `dist/`, rewrites every reference (HTML, manifest, service worker), and derives the service-worker `CACHE_NAME` from the build hash so clients reliably pick up new deploys.

> **Contributors:** see [`HANDOFF.md`](HANDOFF.md) for architecture, conventions, gotchas, and the current project state.

## Run locally

Service workers require http/https, so use a static server (not `file://`):

- **Dev (serve source):** `npx serve -s .` or `python3 -m http.server 8080`, then open the printed URL.
- **Production fidelity (serve the build):** `npm run build` then serve `dist/` (e.g. `npx serve -s dist`). This is what the Docker image ships.

Data persists in your browser storage on this device.

## Run with Docker

Prereq: Docker Desktop (Compose v2).

```bash
docker compose up -d --build   # multi-stage build → serves dist/ via nginx
# open http://localhost:6969
docker compose down
```

The image is multi-stage: `node:22-alpine` runs the build, then `nginx:1.30-alpine` serves `dist/`. Both base images are pinned by tag **and** digest. `nginx.conf` serves fingerprinted assets `immutable` and HTML / service worker / manifest / `/src/` `no-cache`, with security headers (`nginx/security-headers.conf`) applied per-location.

You can run the header/cache smoke test directly:

```bash
bash scripts/smoke-test.sh     # builds image, runs container, asserts cache + security headers
```

## Run the published image (GHCR)

CI publishes a prebuilt image to GitHub Container Registry on every push to `main`:

```bash
docker run --rm -p 6969:80 ghcr.io/johnnyrobot/companero:latest
# open http://localhost:6969
```

> The GHCR package may be private by default — set it public in the repo's **Packages** settings for anonymous `docker pull` to work.

## PWA install
- In Chrome/Edge: look for the install icon in the address bar, or menu → Install App.
- The app works offline after first load.
- App icons (`icons/icon-192.png`, `icons/icon-512.png`, `icons/icon-maskable-512.png`) are generated by `npm run icons` and committed.

## Privacy
- All data stays on your device in `localStorage`, under the keys
  `student_planner.classes.v1` (classes) and `course_companion.profile.v1` (profile).
- Clearing site data, or using a different browser/profile/device, will not carry data over.

## CI
GitHub Actions (`.github/workflows/ci.yml`) runs on pushes/PRs:
1. **test** — install, `npm test`, `npm run build` on Node 22.
2. **build-and-smoke-test** — builds the Docker image and asserts cache + security headers against a running container.
3. **deploy** — on push to `main` (after the above pass), builds and publishes the image to `ghcr.io/johnnyrobot/companero` (`:latest` + commit SHA). No external secrets — uses the built-in `GITHUB_TOKEN`.
